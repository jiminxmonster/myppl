# QA Fix List

작성일: 2026-03-11

## 이번 검수에서 수정 완료

- 한글 게시판 슬러그 진입 실패 수정
  - 원인: Django URL이 `<slug:slug>`라 한글 슬러그를 받지 못함
  - 조치: `backend/apps/boards/urls.py`에서 게시판 상세/글목록 경로를 `<str:slug>`로 변경

- 새 게시판 생성 후 메뉴 미반영 수정
  - 원인: 헤더 메뉴가 하드코딩되어 있었고, 생성 후 동일 탭 갱신 신호가 없었음
  - 조치: 동적 게시판 메뉴 + `community:boards-updated` 이벤트 반영

- 게시판 진입/글쓰기 제목 동기화 수정
  - 원인: 화면 제목에 실제 게시판명 대신 URL slug를 그대로 사용
  - 조치: 게시판 메타 API 추가 후 실제 `board.name` 사용

- 게시글 작성 실패(`No Board matches the given query.`) 수정
  - 원인: 기본 `free`, `notice` 게시판이 숨김 상태로 남아 있었음
  - 조치: 기본 게시판 가시성 데이터 정상화, 부트스트랩 명령에서 기본 게시판 상태 보정 추가

- 핫딜/중고장터 상세 링크 부재 수정
  - 원인: 목록에서 상세 진입 UX가 약했음
  - 조치: 제목 링크와 상세 보기 링크 추가

- 중고장터 상세의 구매 요청 액션 부재 수정
  - 원인: 상세 페이지에는 결제 패널만 있고 구매 요청 UI가 없었음
  - 조치: `frontend/components/marketplace/marketplace-detail-actions.tsx` 추가

- 운영자 신고 처리 시 블라인드 사유가 raw 코드값으로 보이던 문제 수정
  - 원인: `report.reason` 값을 그대로 게시글 블라인드 사유에 저장
  - 조치: `get_reason_display()` 값으로 저장

- 운영자가 블라인드 게시글 원문을 못 보던 문제 수정
  - 원인: 게시글 상세가 SSR 요청만 사용해서 인증 컨텍스트 없이 블라인드 응답을 받았음
  - 조치: hydrate 후 인증 포함 API 재요청으로 최신 상세 반영

- 실시간 알림 WebSocket 미작동 수정
  - 원인: `channels`를 쓰고 있지만 backend 컨테이너가 `gunicorn config.wsgi`로 떠 있었음
  - 조치: `daphne` 추가, backend를 `config.asgi:application`으로 실행
