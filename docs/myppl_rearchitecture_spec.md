# myppl 전면 개편 지시서

## 목표

기존 `핫딜 / 중고장터 / 커뮤니티` 중심 구조를 `구매자`와 `판매자`가 명확히 구분되는 상품 매칭 플랫폼 구조로 재편한다.  
핵심은 아래 세 축이다.

1. `구매자 로그인`과 `판매자 로그인`을 분리한다.
2. `커뮤니티`는 운영자가 켜고 끌 수 있는 `상위 게시판 + 서브 게시판` 구조로 바꾼다.
3. 메인 화면은 운영자가 편집 가능한 `광고 슬라이드 + 카테고리별 인기 상품 슬라이드 + 커뮤니티 최신글` 구조로 정리한다.

## 기본 운영 원칙

1. 회원은 `구매자`와 `판매자` 타입으로 구분한다.
2. 운영자는 별도의 `operator_role`을 유지하고, 회원 타입과 독립적으로 관리한다.
3. 커뮤니티 상위 메뉴는 운영자가 직접 추가/삭제/노출 제어할 수 있어야 한다.
4. 상위 게시판 아래에는 서브 게시판을 둘 수 있다.
5. 메인 페이지의 광고/추천 섹션은 운영자가 편집 가능해야 한다.
6. 판매자가 등록한 상품과 구매자가 등록한 `원하는 상품`은 이후 매칭 가능한 구조여야 한다.

## 로그인 및 계정 정책

### 공통

- 기본 계정 예시
  - 판매자: `sell / sell`
  - 구매자: `buy / buy`
- 공통 메뉴
  - 상품리스트
  - 구매자커뮤니티
  - 판매자커뮤니티
  - 공지
  - 알림
  - 개인프로필

### 판매자 로그인

- 추가 메뉴
  - `내판매상품`
- 주요 액션
  - `상품올리기+`
- 개인프로필
  - 프로필사진
  - 아이디
  - 이름
  - 로그아웃
- 판매자는 직접 상품을 등록하고 수정/관리할 수 있어야 한다.

### 구매자 로그인

- 추가 메뉴
  - `원하는상품`
- 주요 액션
  - `원하는상품+`
- 개인프로필
  - 프로필사진
  - 아이디
  - 이름
  - 로그아웃
- 구매자는 원하는 상품 사양과 알림 설정을 저장할 수 있어야 한다.

## 커뮤니티 구조

### 기본 노출 규칙

- 기본 상위 게시판은 아래 두 개만 노출한다.
  - 구매자커뮤니티
  - 판매자커뮤니티
- 두 게시판의 초기 상태는 `서브 게시판 없음`이다.

### 운영자 기능

- 상위 게시판 생성/수정/삭제/숨김
- 서브 게시판 생성/수정/삭제/숨김
- 탑 메뉴 노출 여부 설정
- 정렬 순서 설정

### 사용자 노출 규칙

- 상위 게시판에 서브 게시판이 있으면 hover 또는 클릭 시 서브 게시판 메뉴를 보여준다.
- 서브 게시판이 없으면 상위 게시판 클릭 시 바로 해당 커뮤니티 목록으로 이동한다.

## 메인 화면 구성

### 1. 탑 메뉴

- 상품리스트
- 구매자커뮤니티
- 판매자커뮤니티
- 공지
- 알림
- 역할별 메뉴
  - 판매자: 내판매상품
  - 구매자: 원하는상품
- 개인프로필

### 2. 광고 슬라이드

- 기업 상품광고 배너를 자동 슬라이드로 노출한다.
- 기본 전환 주기: `3초`
- 운영자 설정에서 편집 가능해야 한다.

### 3. 카테고리별 인기 상품 슬라이드

- 예시
  - 가전제품 조회수 높은순
  - 패션제품 조회수 높은순
  - 식품류 조회수 높은순
- 운영자 설정에서 섹션 추가/삭제/정렬 가능해야 한다.

### 4. 커뮤니티 최신글 영역

- 최근 구매자커뮤니티 최신글
- 최근 판매자커뮤니티 최신글
- 각 섹션을 분리해 메인 화면 하단에 노출한다.

## 판매자 상품 등록

- 판매자는 `상품올리기+`를 통해 상품을 등록한다.
- 등록된 상품은
  - 상품리스트
  - 내판매상품
  두 영역에서 확인 가능해야 한다.
- 수정/편집 가능해야 한다.
- 상세 옵션 입력 방식
  - 운영자가 미리 만든 체크리스트 사용
  - 판매자가 직접 세부 체크리스트 추가 가능
  - 향후 외부 API/크롤링 결과와 연결 가능

## 구매자 원하는상품 등록

- 구매자는 `원하는상품+`를 통해 원하는 상품 조건을 등록한다.
- 조건은 아래 두 소스에서 가져올 수 있어야 한다.
  - 운영자가 만든 표준 옵션
  - 판매자가 실제로 사용한 옵션 후보
- 알림 설정
  - 내부 알림
  - 카카오 알림
  - 문자
  - 이메일
- 저장 후 `원하는상품` 목록에서 수정/편집 가능해야 한다.

## 매칭 구조

- 판매자 상품 등록 정보와 구매자 원하는상품 조건은 이후 매칭 가능한 구조로 저장한다.
- 매칭 기준
  - 카테고리
  - 세부 옵션
  - 키워드
  - 가격 범위
- 향후 이벤트
  - 신규 등록 상품 매칭
  - 가격 변동
  - 재입고 또는 조건 충족

## 1차 구현 범위

1. 회원 타입 `buyer / seller` 추가
2. 기본 계정 `sell / sell`, `buy / buy` 생성
3. 커뮤니티 상위 게시판/서브 게시판 구조 추가
4. 기본 커뮤니티 탑 메뉴
   - 구매자커뮤니티
   - 판매자커뮤니티
5. 헤더 메뉴를 역할 기반으로 분기
6. 메인 화면을 슬라이드/카테고리/최신글 구조로 재구성
7. 판매자용 `내판매상품`
8. 구매자용 `원하는상품`
9. 운영자용 커뮤니티 구조 편집 UI

## 2차 구현 범위

1. 운영자 광고 배너 관리
2. 운영자 메인 섹션 편집
3. 판매자 체크리스트 확장
4. 구매자 원하는상품 체크리스트 저장
5. 매칭 엔진
6. 채널별 실제 알림 발송기
