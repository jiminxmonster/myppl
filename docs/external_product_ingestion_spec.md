# 외부 쇼핑몰 상품수집 및 필터관리 요구사항

## 목표

사용자가 원하는 상품군과 세부 조건을 선택하면, 시스템이 외부 쇼핑몰 API 또는 제휴 데이터 소스에서 관련 상품 정보를 수집하고 내부 DB에 저장한 뒤 빠르게 검색·비교·알림에 활용할 수 있어야 한다.

## 핵심 운영 원칙

1. 사용자에게 노출되는 카테고리와 체크 필터는 내부 표준 구조를 기준으로 한다.
2. 외부 쇼핑몰의 카테고리와 속성은 자동으로 수집할 수 있어야 한다.
3. 자동 수집된 외부 카테고리/속성은 그대로 노출하지 않고, 관리자 승인 또는 정제를 거친 뒤 사용자 화면에 반영한다.
4. 운영 방식은 `완전 수동`이 아니라 `자동 수집 + 관리자 승인 + 내부 표준화`를 기본으로 한다.

## 데이터 흐름

1. 외부 쇼핑몰 API 연동 정보를 등록한다.
2. 외부 쇼핑몰의 카테고리와 속성 후보를 자동 수집한다.
3. 수집한 후보를 내부 표준 카테고리/필터 체계에 매핑한다.
4. 관리자가 승인, 숨김, 병합, 이름수정, 값정규화를 수행한다.
5. 승인된 구조만 사용자 화면에 노출한다.
6. 사용자가 선택한 조건을 기준으로 상품 수집/검색/알림 엔진이 동작한다.
7. 조건에 맞는 신규 상품, 가격 하락, 재입고 이벤트가 발생하면 사용자에게 선택한 채널로 알림을 발송한다.

## 관리자 작업 항목

### 1. 외부 연동 관리

- 외부 쇼핑몰 제공자 등록
- API 키/시크릿/엔드포인트 관리
- 연동 활성/비활성
- 마지막 동기화 시각 확인

### 2. 내부 표준 카테고리 관리

- 상품 대분류 생성
- 세부 카테고리 생성
- 사용자 노출 여부 관리
- 정렬 순서 관리

### 3. 내부 표준 필터 관리

- 카테고리별 필터 생성
- 체크박스/단일선택/범위형 필터 정의
- 필터명, 정렬순서, 노출 여부 관리
- 필터 옵션 수동 추가/수정/삭제

### 4. 자동 수집 후보 승인

- 외부 카테고리 후보 목록 확인
- 내부 카테고리에 연결
- 승인/기각/보류 처리
- 외부 속성 후보를 내부 필터에 연결
- 동일 의미 값 병합
  - 예: `네이비`, `남색`, `dark navy`

### 5. 사용자 관심조건 및 알림 관리

- 사용자가 카테고리/필터 조합을 저장할 수 있어야 한다.
- 관심조건별 알림 활성/비활성 설정이 가능해야 한다.
- 알림 대상 이벤트는 최소 아래 항목을 지원해야 한다.
  - 신규 상품 발견
  - 가격 하락
  - 재입고
- 알림 채널은 최소 아래 항목을 지원해야 한다.
  - 내부 알림/쪽지
  - 이메일
  - 카카오톡
  - 문자
- 사용자는 채널을 복수 선택할 수 있어야 한다.
- 발송 이력과 성공/실패 상태가 저장되어야 한다.
- 중복 발송 방지 규칙이 있어야 한다.

## 구현 원칙

### 1. 내부 표준 구조와 외부 원본 구조를 분리

- 내부 구조: 사용자 UI와 검색의 기준
- 외부 구조: 원본 카테고리/속성/값 저장용

### 2. 승인 상태를 명확히 관리

- `pending`
- `approved`
- `rejected`

### 3. 정규화 가능한 값을 별도 보관

- 외부 원본명
- 내부 정규화명
- 원본 payload
